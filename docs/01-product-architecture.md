# 产品架构

## 产品定位

NextNews 是一个可运营的新闻与热榜聚合系统，而不是单纯的热榜阅读器。它需要同时支持：

- 公开前台，展示站点、分类、频道、榜单和历史内容。
- 用户个人中心，支持订阅频道和查看个性化动态。
- 私有后台，控制前台展示、数据源、抓取、历史快照和运营模块。
- 抓取系统，定时请求第三方站点并将结果入库。

## 系统分层

```txt
用户前台
  首页 / 站点 / 分类 / 频道 / 日报 / 动态 / 追踪 / 榜中榜 / 话题 / 个人中心

后台管理
  站点 / 分类 / 频道 / 抓取 / 内容库 / 历史快照 / 前台配置 / 用户订阅 / 系统设置

服务层
  前台查询 / 后台管理 / 抓取调度 / 快照入库 / 聚合计算 / 订阅动态 / 话题生成

数据层
  PostgreSQL / Redis / 搜索服务（后续）
```

## 前台模块

### 首页

首页由后台配置，不在代码中写死。首页可以包含：

- 热门站点横向入口。
- 推荐频道卡片。
- 实时热榜。
- 分类区块。
- 榜中榜入口。
- 日报入口。
- 话题入口。

### 站点页

展示某个站点下的所有频道，例如微博下可以有热搜、文娱榜、要闻榜等。站点页重点展示来源品牌和频道集合。

### 分类页

展示某个分类下的频道，例如科技分类可以包含 GitHub Trending、少数派、IT之家、Product Hunt 等频道。

### 频道页

展示某个频道的最新快照、历史快照、更新时间和订阅按钮。频道页是历史功能和订阅功能的核心入口。

### 日报

日报是基于一段时间内的快照数据生成的内容集合，可以自动生成，也可以后台审核后发布。

### 动态

动态是用户订阅频道后的个性化信息流。它不是全站最新，而是按用户订阅范围聚合。

### 追踪

追踪是用户配置关键词、话题或站点后产生的命中流。它依赖已入库内容，不直接实时抓取。

### 榜中榜

榜中榜是平台自计算的跨频道聚合榜单。它需要后台配置参与频道、权重、时间范围和计算规则。

### 话题

话题用于聚合同一事件在多个频道中的出现情况。话题可以由算法生成，也可以后台人工合并、隐藏或置顶。

### 个人中心

个人中心第一版需要支持：

- 订阅频道。
- 管理订阅。
- 查看订阅动态。
- 管理追踪规则。

收藏、阅读历史、会员权益可以后续扩展。

## 后台定位

后台只给项目拥有者使用，但它不是简单的数据库维护工具。后台的目标是控制前台和数据流：

- 控制哪些站点、分类、频道显示。
- 控制首页和导航。
- 控制频道是否允许订阅。
- 控制抓取频率和抓取状态。
- 查看抓取失败和数据异常。
- 管理历史快照和内容屏蔽。
- 配置日报、榜中榜和话题。

## MVP 范围

第一阶段建议只做这些：

- 站点管理。
- 分类管理。
- 频道管理。
- 抓取任务和日志。
- 最新快照入库。
- 首页配置。
- 前台首页、频道页、个人订阅。

第二阶段再做：

- 历史快照查询。
- 动态。
- 日报。
- 榜中榜。

第三阶段再做：

- 话题聚合。
- 追踪。
- 会员历史权限。
- 搜索服务。
