# 运营增强能力

本阶段补齐 5 个围绕运营和商业化的能力：会员历史权限、订阅通知、增强搜索、运营统计、日报人工精选。它们都建立在“采集后入库、前台读取数据库”的主线之上，不引入前台实时请求第三方站点。

## 会员历史权限

新增 `user_membership` 表，用于后台给前台用户授予权益。

关键字段：

- `user_id`：对应 `sys_user`
- `plan_key`：套餐 key，例如 `free`、`member`
- `plan_name`：后台显示名称
- `status`：`active`、`expired`、`canceled`
- `history_days`：可查看最近多少天历史
- `expires_at`：到期时间

当前规则：

- 访客默认可看最近 7 天历史
- 登录普通用户默认可看最近 30 天历史
- 后台授予会员后按 `history_days` 控制历史范围
- 频道历史页和快照详情页都会按最早可访问日期过滤
- 日报详情也会按同一历史权益边界校验

后台入口：

- `/admin/users`

## 订阅频道通知

`rel_user_channel_subscription.notify_enabled` 已正式启用。

当前流程：

1. 用户在个人中心对已订阅频道开启或关闭通知。
2. Worker 采集频道并产生新快照。
3. 系统查找该频道下开启通知的订阅用户。
4. 为每个用户写入 `user_notification`，类型为 `channel_update`。
5. 通知链接指向对应频道快照页。

重复控制：

- 同一个用户、同一个快照只生成一条订阅通知。

前台入口：

- `/account`
- `/notifications`

## 增强搜索

`/search` 从基础关键词搜索升级为组合筛选。

支持条件：

- 关键词
- 站点
- 频道
- 开始日期
- 结束日期

结果能力：

- 检索标题、摘要、标签、频道名和站点名
- 过滤被屏蔽内容
- 按快照时间倒序展示
- 高亮命中的关键词

后续可继续增强：

- PostgreSQL `pg_trgm` 或全文索引
- 搜索日志
- 热门搜索词
- 独立搜索服务

## 运营统计

新增后台运营统计页，用于快速判断系统运行情况。

后台入口：

- `/admin/operations/analytics`

当前指标：

- 活跃频道数
- 首页频道数
- 今日快照数
- 今日内容数
- 今日失败任务数
- 用户数
- 频道订阅数
- 有效会员数
- 今日通知数
- 未读通知数
- 今日追踪命中数
- 已发布日报数

频道健康表：

- 站点 / 频道
- 频道状态
- 是否开启采集
- 最近成功时间
- 最新快照条数
- 今日失败次数

## 日报人工精选

新增 `biz_daily_report_item` 表，用于把具体快照条目人工挂载到某一份日报。

关键字段：

- `report_id`：对应 `biz_daily_report`
- `snapshot_item_id`：对应 `biz_snapshot_item`
- `sort`：展示排序
- `note`：推荐语
- `created_by`：操作管理员

后台入口：

- `/admin/operations/daily`
- `/admin/operations/daily/[id]`

前台展示：

- `/daily/[reportDate]` 会优先展示“人工精选”
- 自动聚合的频道内容仍保留在下方

## 开发顺序状态

这一阶段完成后，NextNews 已经具备更完整的运营闭环：

```txt
采集入库
  -> 历史快照
  -> 搜索 / 日报 / 榜中榜 / 话题
  -> 用户订阅 / 追踪 / 通知
  -> 会员历史权限
  -> 后台运营统计
```

下一阶段建议继续做：

- 商业化总开关、会员套餐和订单预留
- 搜索日志、热门关键词分析和搜索质量观察
- 收藏、阅读历史和统一 `/go` 跳转记录
- 日报模板、任务队列预留和后台操作日志
- 运营统计导出

上述能力已经在下一份文档 [商业化与运营控制](./10-commercialization-and-ops.md) 中落地为首版实现。后续继续扩展时，重点转向支付闭环、任务队列消费、搜索索引优化、通知策略细化和运营趋势图。
