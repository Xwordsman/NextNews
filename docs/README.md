# NextNews Docs

NextNews 是一个带后台运营能力的新闻与热榜聚合系统。它参考 Newsnow 的轻量阅读体验，也参考 TopHub 的站点、频道、订阅、历史快照和榜单体系。

本文档目录用于沉淀产品、架构和开发约定。当前阶段已经进入功能闭环开发，所有新增功能都需要同步更新这里的产品边界、数据模型和开发顺序。

## 已确定方向

- 前端框架：Next.js 16.2.9。
- 数据库：PostgreSQL。
- 后台路径：`/admin`。
- 后台使用者：仅项目拥有者本人。
- 前台能力：用户可以注册/登录，并在个人中心订阅频道。
- 数据模式：第三方站点数据由抓取系统入库，前台不直接实时请求第三方榜单。
- 部署方式：第一阶段以服务器 Docker Compose 部署为主，适配宝塔面板的 Docker 容器编排。
- 核心能力：站点、分类、频道、历史快照、首页配置、日报、动态、追踪、榜中榜、话题。

## 核心概念

### 站点 Site

内容来源品牌，例如知乎、微博、GitHub、Bilibili。站点本身不一定直接抓取，站点下面可以挂多个频道。

### 频道 Channel

一个具体可抓取的榜单或信息流，例如知乎热榜、微博热搜、GitHub Trending Today。频道属于某个站点，也可以归入一个或多个分类。

### 频道定义 Channel Definition

一个代码仓库中的频道定义文件，负责声明频道如何采集数据。RSS 和复杂爬虫都统一使用频道定义文件。后台只绑定频道定义，不在线编写或执行任意代码。

### 分类 Category

前台组织频道的方式，例如科技、财经、娱乐、社区、AI。分类用于导航、首页区块和订阅筛选。

### 快照 Snapshot

某个频道在某次抓取时的完整榜单状态。历史查询、按月按年查看、榜中榜、日报和话题都依赖快照。

### 条目 Item

快照中的单条内容，包含标题、URL、图片、摘要、排名、热度、标签和扩展字段。

### 前台模块 Page Block

由后台控制的展示单元，例如首页热门站点、实时热榜、分类频道、日报入口、话题列表、榜中榜模块。

## 设计原则

- 后台控制前台，前台不硬编码展示规则。
- 抓取入库优先，实时请求第三方只作为后台手动调试能力。
- 频道是最重要的运营对象，站点和分类围绕频道服务。
- 频道定义文件负责“怎么采集”，后台数据库负责“怎么运营和展示”。
- 历史快照从第一版就保留，为未来会员能力预留空间。
- 采集代码放在代码仓库，后台只管理开关、频率、展示、分类和调度，不提供在线编写采集代码能力。
- 数据库负责结构化存储和历史查询，全文搜索和语义搜索后续接独立搜索服务。
- Web 服务负责页面、后台和 API；Worker 服务负责定时抓取、快照入库和派生任务，二者使用同一份代码镜像但以不同启动命令运行。

## 开发优先级

开发时不要先做完整前台页面，也不要先把所有后台菜单铺开。第一阶段应该先完成可运行闭环：

```txt
项目骨架 -> 数据库模型 -> 后台基础资料 -> 频道定义 -> Worker 抓取入库 -> 首页读取数据库 -> Docker 部署
```

详细顺序见 [开发顺序](./08-development-roadmap.md)。
