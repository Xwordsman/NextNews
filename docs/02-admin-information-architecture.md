# 后台信息架构

后台路径为 `/admin`。后台只给项目拥有者使用，因此不需要一开始设计复杂 RBAC，但仍然需要登录保护和管理员权限判断。

## 工作台

工作台用于快速判断系统是否健康。

核心指标：

- 今日抓取次数。
- 今日入库条目数。
- 今日失败次数。
- 异常频道数量。
- 最新成功抓取时间。
- 待处理内容屏蔽数。
- 当前启用频道数。
- 当前首页展示频道数。

## 内容源

### 站点管理

站点是内容来源品牌，不等于具体榜单。

字段重点：

- 站点名称。
- 站点 slug。
- 官网 URL。
- Logo。
- 简介。
- 状态。
- 排序。
- 是否前台展示。

### 分类管理

分类用于前台导航、首页模块和频道归类。

字段重点：

- 分类名称。
- 分类 slug。
- 图标。
- 颜色。
- 父级分类。
- 排序。
- 是否导航显示。
- 是否首页显示。

### 频道管理

频道是后台最重要的运营对象。

字段重点：

- 所属站点。
- 频道名称。
- 频道 slug。
- 所属分类。
- 频道定义 key。
- 抓取间隔。
- 是否启用抓取。
- 是否前台公开。
- 是否首页显示。
- 是否允许订阅。
- 展示样式。
- 权重。
- 排序。

频道管理只负责运营配置，不负责在线编写采集代码。新增频道时，后台绑定代码仓库中已经存在的频道定义文件。

推荐新增方式：

- 从已支持频道添加：后台从频道定义注册表中读取可用频道，选择后创建后台频道记录。
- 添加 RSS 频道：RSS 也需要生成一个频道定义文件，后台绑定该定义后启用。
- 提交待接入频道：只填写名称、页面地址、期望分类和备注，等待后续开发频道定义文件。

后台不向普通管理界面暴露 JSON path、CSS selector、任意 JS 代码等技术配置。

## 抓取中心

### 抓取任务

用于查看和控制频道抓取。

能力：

- 查看下一次抓取时间。
- 手动触发抓取。
- 暂停频道抓取。
- 恢复频道抓取。
- 查看最近一次抓取结果。

### 抓取日志

记录每次抓取运行结果。

展示内容：

- 频道。
- 开始时间。
- 结束时间。
- 状态。
- 耗时。
- 入库条数。
- 错误信息。

### 失败记录

失败记录用于集中处理异常频道。

常见操作：

- 重试。
- 暂停频道。
- 查看错误堆栈。
- 标记已处理。

## 内容库

### 最新内容

查看各频道最新入库内容。用于人工检查抓取是否正常。

### 历史快照

按站点、频道、日期、月份、年份查看历史榜单。这个页面是未来会员历史能力的基础。

### 屏蔽内容

用于管理被隐藏的标题、URL、关键词或来源。

## 前台运营

### 首页配置

首页配置应该用模块化方式管理。

模块示例：

- 热门站点。
- 实时热榜。
- 分类频道。
- 日报入口。
- 榜中榜。
- 话题。

每个模块需要支持：

- 是否启用。
- 标题。
- 副标题。
- 数据来源。
- 排序。
- 展示数量。
- 展示样式。

### 导航配置

控制前台导航显示哪些页面、分类或频道。

### 热门站点

控制首页横向站点入口。站点入口可以指向站点，也可以指向重点频道。

### 榜中榜配置

控制参与聚合的频道、权重、时间范围和排序规则。

### 日报配置

控制日报生成范围、生成时间、是否自动发布、是否需要审核。

### 话题配置

控制话题显示、隐藏、合并、置顶和推荐。

## 用户中心

### 用户列表

查看前台用户，不做复杂管理。

### 频道订阅

查看用户订阅了哪些频道，也可以用于判断频道受欢迎程度。

### 追踪规则

查看用户配置的关键词、站点、频道或话题追踪规则，并查看追踪命中记录。

### 站内通知

查看用户站内通知，包括追踪命中通知和后续扩展的订阅提醒、系统消息。

## 系统设置

系统设置只放底层参数，不放日常运营项。

包括：

- 站点基础信息。
- 抓取并发。
- 默认抓取间隔。
- 缓存 TTL。
- Redis 配置。
- 管理员账号。
- 操作日志。
